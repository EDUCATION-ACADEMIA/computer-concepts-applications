Test method for testing each component of the system separately?

(A) Integration testing

(B) System testing

(C) ** Unit testing

(D) Acceptance test

EXPLANATIONS BELOW

Concept note-1: -Component testing is defined as a software testing type, in which the testing is performed on each individual component separately without integrating with other components. Itâ€™s also referred to as Module Testing when it is viewed from an architecture perspective.

Concept note-2: -Definition: Component testing, also known as program or module testing, is done after unit testing. In this type of testing those test objects can be tested independently as a component without integrating with other components e.g. modules, classes, objects, and programs.

Concept note-3: -Suppose, in an application there are 5 components. Testing of each 5 components separately and efficiently is called as component testing. Component testing is also known as module and program testing. It finds the defects in the module and verifies the functioning of software.

Concept note-4: -White box testing approach used for unit testing and usually done by the developers.
