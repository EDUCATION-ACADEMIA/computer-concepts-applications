SYSTEMS DEVELOPMENT ANALYSIS

WHY IS SYSTEM ANALYSIS IMPORTANT

[SOURCES]
SSADM may be good choice of methodology because it has:

(A) A view to communicating with the team

(B) ** A process, time and data view

(C) A data, filed and record view

(D) A view to collecting part of the requirements

EXPLANATIONS BELOW

Concept note-1: -SSADM is applicable to projects with technical and business requirements that are unlikely to change. It uses a top-down waterfall project management approach to information systems projects. SSADM isn’t suitable for projects in a volatile business environment, so Agile should be used instead.

Concept note-2: -SSADM combines three methods, complementing each other within a systems development cycle: Logical Data Modelling, Data Flow Modelling, Entity Event Modelling. Its main benefits are: Improvement of quality, detailed documentation of the development stages, and reusability for similar projects that follow.

Concept note-3: -The SSADM is an open methodology based on the waterfall model. It has been used by many commercial businesses, consultants, educational establishments and CASE tool developers.

Concept note-4: -These three are:-• LOGICAL DATA MODELS (LDS) • DATAFLOW MODELS (DFD) • ENTITY LIFE HISTORIES (ELH) Initially this section reviews each technique, then moves on to show how these techniques are interrelated, and how they can be used to cross-check and validate each other.

Concept note-5: -Advantages of SSAD: Timelines: as mentioned before, SSAD can be used to improve the way a project is controlled and directed. This is due to the fact that it allows one to plan the project well which is essential to deliver the product on time.
