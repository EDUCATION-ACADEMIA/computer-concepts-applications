APPLICATION DEVELOPMENT PROCESSES

[SOURCES]
It is a type of DFD that can either be represented as a rectangle with absence of both smaller sides or as an open sided rectangle with only one side missing.

(A) Entities

(B) Process

(C) ** Data Storage

(D) Data Flow

EXPLANATIONS BELOW

Concept note-1: -Data Storage – There are two variants of data storage – it can either be represented as a rectangle with absence of both smaller sides or as an open-sided rectangle with only one side missing. Data Flow – Movement of data is shown by pointed arrows.

Concept note-2: -Data flow diagram notations For example, Yourdon and Coad notation uses circles to represent processes, whereas Gane and Sarson notation use rectangles with rounded corners.

Concept note-3: -Gane and Sarson: Squares or rectangles denote external entities. Open-ended rectangles symbolize data storage. Arrows indicate data flow.

Concept note-4: -The tasks performed on the data is known as process. Process is represented by circle. Somewhere round edge rectangles are also used to represent process.

Concept note-5: -There are two distinct types of data flow diagrams: the logical DFD and the physical DFD.
