Why can a file name not contain forward and backward slashes?

(A) ** Windows recognizes these as parts of a file path

(B) Windows doesnâ€™t like slashes

(C) File names must contain only letters

(D) Slashes designate different hard drives

EXPLANATIONS BELOW

Concept note-1: -You cannot use a backslash in the name for the actual file or directory because it is a reserved character that separates the names into components.

Concept note-2: -The reason Microsoft is backwards on this goes back to MS-DOS 2.0 (DOS 1.0 had no directory hierarchy), which used a backslash to stay compatible with Dos 1.0 commands, which used slash for command line switches.

Concept note-3: -MS-DOS 2.0 copied the hierarchical file system from Unix and thus used the forward slash, but (possibly on the insistence of IBM) added the backslash to allow paths to be typed into the command shell while retaining compatibility with MS-DOS 1.0 and CP/M where the slash was the command-line option indicator.

Concept note-4: -These characters are prevented from being used in file names because it would confuse the system when trying to access the file.
