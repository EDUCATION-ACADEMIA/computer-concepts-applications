OPERATING SYSTEMS FOR COMPUTERS

COMPUTER OPERATING SYSTEMS

[SOURCES]
when a process is saved in virtual memory it is suspended. When loaded back into RAM for processing it is restored

(A) cache memory

(B) ** RAM

(C) ROM

(D) permanent memory

EXPLANATIONS BELOW

Concept note-1: -If your system has to rely too heavily on virtual memory, you will notice a significant performance drop. The key is to have enough RAM to handle everything you tend to work on simultaneously–then, the only time you “feel” the slowness of virtual memory is is when there’s a slight pause when you’re changing tasks.

Concept note-2: -If that data is needed again, the computer’s MMU will use a context switch to resume execution. While copying virtual memory into physical memory, the OS divides memory with a fixed number of addresses into either pagefiles or swap files.

Concept note-3: -Virtual memory is an area of a computer system’s secondary memory storage space, such as an HDD or SSD, that acts as if it were a part of the system’s RAM or primary memory. Ideally, the data needed to run applications is stored in RAM, where the CPU can quickly access it.

Concept note-4: -Once the operating system is loaded, the boot loader transfers control to it and is no longer needed. The operating system will initialize itself, configure the system hardware (e.g., set up memory management, set timers, set interrupts), and load device drivers, if needed.
