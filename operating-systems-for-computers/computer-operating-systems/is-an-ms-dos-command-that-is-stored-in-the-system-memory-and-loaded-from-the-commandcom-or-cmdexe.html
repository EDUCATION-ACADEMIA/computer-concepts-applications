COMPUTER OPERATING SYSTEMS

is an MS-DOS command that is stored in the system memory and loaded from the command.com or cmd.exe.

(A) ** Internal Command

(B) Cache

(C) Foreground Task

(D) Device Driver

(E) CLI (command line interface)

EXPLANATIONS BELOW

Concept note-1: -An internal command is the MS-DOS command stored in system memory and loaded from command.com or cmd.exe. Internal commands include MD, DATE, COPY, CD, TIME, COPR CON, TYPE, etc.

Concept note-2: -Answer. CONFIG. SYS file in MS-DOS contains internal commands that are loaded during Booting process.

Concept note-3: -In MS-DOS, there are two ways commands are executed: internally and externally. An internal command is embedded into the COMMAND.COM file, and an external command is not and requires a separate file to operate.

Concept note-4: -The Windows CMD shell CMD.exe contains a number of ‘internal’ commands, additional ‘external’ commands are also supplied as separate executable files. External commands are generally stored in the C:32 folder, this folder is part of the system PATH .

Concept note-5: -With copy con command If you’re running an MS-DOS version 4. x or lower or you cannot use the edit or the start method you can also use copy con to create a file, as shown below. Upon executing the above command, the cursor moves down one line to a blank line, allowing you to create the new file line by line.
