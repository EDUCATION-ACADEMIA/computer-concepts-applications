In HDFS Namenode and Datanode works in a ____ fashion.

(A) worker-master fashion

(B) ** master-slave fashion

(C) master-worker fashion

(D) slave-master fashion

EXPLANATIONS BELOW

Concept note-1: -HDFS – Hadoop Distributed File System is the storage layer of Hadoop. It is most reliable storage system on the planet. HDFS works in master-slave fashion, NameNode is the master daemon which runs on the master node, DataNode is the slave daemon which runs on the slave node.

Concept note-2: -HDFS works in a Master –Slave fashion.In which it have one master node that is known as Name data(It manage the whole system), no of slave mode is known as Data nodes. HDFS is fault tolerant and useful to design at low cost Hardware.

Concept note-3: -The NameNode executes file system namespace operations like opening, closing, and renaming files and directories. It also determines the mapping of blocks to DataNodes. The DataNodes are responsible for serving read and write requests from the file system’s clients.

Concept note-4: -The main difference between NameNode and DataNode in Hadoop is that the NameNode is the master node in Hadoop Distributed File System (HDFS) that manages the file system metadata while the DataNode is a slave node in Hadoop distributed file system that stores the actual data as instructed by the NameNode.

Concept note-5: -NameNode is used when the Primary NameNode goes down. Answer : Standby.
