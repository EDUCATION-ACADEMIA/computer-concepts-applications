BASICS OF BIG DATA

[SOURCES]
A distributed data storage system that is intended to manage structured data and is designed as a reliable system for managing data on a petabytes scale and in thousands of machines (computers) is a technology:

(A) ** Google Bigtable

(B) Google MapReduce

(C) Google File System (GFS)

(D) Google Play Store

(E) Google Super Data

EXPLANATIONS BELOW

Concept note-1: -Bigtable is a distributed storage system for managing structured data that is designed to scale to a very large size: petabytes of data across thousands of commodity servers. Many projects at Google store data in Bigtable, including web indexing, Google Earth, and Google Fi-nance.

Concept note-2: -Bigtable is a distributed storage system that is structured as a large table: one that may be petabytes in size and distributed among tens of thousands of machines.

Concept note-3: -Use Cloud Bigtable as the storage engine that grows with you from your first gigabyte to petabyte-scale for low-latency applications as well as high-throughput data processing and analytics.

Concept note-4: -The database was designed to be deployed on clustered systems and uses a simple data model that Google has described as “a sparse, distributed, persistent multidimensional sorted map.” Data is assembled in order by row key, and indexing of the map is arranged according to row, column keys and timestamps.

Concept note-5: -Bigtable is ideal for storing large amounts of single-keyed data with low latency. It supports high read and write throughput at low latency, and it’s an ideal data source for MapReduce operations.
