TYPES OF COMPUTER SECURITY

[SOURCES]
Which of the following describes the concept of RPO?​

(A) Recovery Point Occurrence describes the the data that is logged when a failure occurs​

(B) Recovery Point Operation describes the work required to recovery from a failure​

(C) ** Recovery Point Objective describes the acceptable amount of data loss over a given time​

(D) Recovery Point Objective describes a point in time at which a disaster occurs​

EXPLANATIONS BELOW

Concept note-1: -Explanation. The Recovery Point Objective (RPO) describes the acceptable amount of data loss measured in time. It is the point in time to which data must be recovered as defined by the organization. The RPO is generally a definition of what an organization determines is an “acceptable loss” in a disaster situation.

Concept note-2: -Recovery point objective (RPO) is defined as the maximum amount of data – as measured by time – that can be lost after a recovery from a disaster, failure, or comparable event before data loss will exceed what is acceptable to an organization.

Concept note-3: -Recovery point objective (RPO) is the maximum acceptable amount of data loss after an unplanned data-loss incident, expressed as an amount of time.

Concept note-4: -As a quick refresher, RTO stands for Recovery Time Objective and is a measure of how quickly after an outage an application must be available again. RPO, or Recovery Point Objective, refers to how much data loss your application can tolerate.

Concept note-5: -The RPO is expressed backward in time–that is, into the past–from the instant at which the failure occurs and can be specified in seconds, minutes, hours or days. It is an important consideration in a disaster recovery plan (DRP).
