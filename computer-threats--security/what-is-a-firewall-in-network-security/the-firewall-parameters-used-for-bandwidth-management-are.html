WHAT IS A FIREWALL IN NETWORK SECURITY

[SOURCES]
The firewall parameters used for bandwidth management are

(A) Protocols and ports

(B) Layer 7 Protocol

(C) Address list

(D) content

(E) ** mangle

EXPLANATIONS BELOW

Concept note-1: -Mangle is a kind of ‘marker’ that marks packets for future processing with special marks. Many other facilities in RouterOS make use of these marks, e.g. queue trees, NAT, routing. They identify a packet based on its mark and process it accordingly.

Concept note-2: -Bandwidth management means giving a guaranteed minimum portion of the available bandwidth to some types of traffic. It also sets limits for how much of the total available bandwidth each type of traffic is allowed to consume at any given time. You can set a limit, a guarantee, or both for any given type of traffic.

Concept note-3: -To create a mangle rule for upload packets, go to IP>firewall>mangle>add. Set chain to forward, select your LAN interface as the in interface, click on advance and choose the address-list created in step 2 as your source address-list. Click on action and select “mark packet” and enter a name for the packet mark.

Concept note-4: -Priority of packets can be set using action=set-priority of IP firewall mangle rules or bridge filter/nat rules. Priority can be set to a specific value or taken from the ingress priority using the from-ingress setting. Ingress priority is the priority value that was detected on the incoming packet, if available.
