EXPLANATIONS BELOW

Concept note-1: -An interpreter works more or less similar to a compiler. The only difference between their working is that the interpreter does not generate any intermediate code forms, reads the program line to line checking for errors, and runs the program simultaneously.

Concept note-2: -For interpreted languages, however, a syntax error may be detected during program execution, and an interpreter’s error messages might not differentiate syntax errors from errors of other kinds. There is some disagreement as to just what errors are “syntax errors".

Concept note-3: -An interpreter is a program that directly executes the instructions in a high-level language, without converting it into machine code. In programming, we can execute a program in two ways. Firstly, through compilation and secondly, through an interpreter. The common way is to use a compiler.

Concept note-4: -An interpreter is a general purpose language providing very efficient execution.
