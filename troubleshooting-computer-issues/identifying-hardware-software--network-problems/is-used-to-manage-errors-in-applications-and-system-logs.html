TROUBLESHOOTING COMPUTER ISSUES

IDENTIFYING HARDWARE SOFTWARE NETWORK PROBLEMS

[SOURCES]
Is used to manage errors in applications and system logs

(A) Solution Database

(B) ** Event viewer

(C) Task manager

(D) Antivirus

EXPLANATIONS BELOW

Concept note-1: -Microsoft Windows Server Event Viewer is a monitoring tool that shows a log of events that can be used to troubleshoot issues on a Windows-based system. The Event Viewer displays information about application, security-related, system and setup events.

Concept note-2: -On the Search bar, type Event Viewer, and then select the Event Viewer desktop app. In Event Viewer, expand the Windows Logs folder, and select the Application event log.

Concept note-3: -Windows event log is an in-depth record of events related to the system, security, and application stored on a Windows operating system. Event logs can be used to track system and some application issues and forecast future problems.

Concept note-4: -The Windows Applications and Services Log enables you to monitor the Applications and Services Logs on Windows devices, similar to the Windows Event Log Service. Since you can assign up to 1, 000 instances of this service to a device, you can give each instance a Service Identifier.

Concept note-5: -Open “Event Viewer” by clicking the “Start” button. Click “Control Panel” > “System and Security” > “Administrative Tools", and then double-click “Event Viewer” Click to expand “Windows Logs” in the left pane, and then select “Application". Click the “Action” menu and select “Save All Events As".
